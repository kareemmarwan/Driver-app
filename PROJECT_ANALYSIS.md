# تحليل مشروع "دري فري - Dreefree"

## 📋 فكرة المشروع

**تطبيق توصيل متكامل** يربط العملاء بمقدمي خدمات التوصيل في فلسطين (غزة بشكل أساسي). يتيح التطبيق للمستخدمين:
- طلب توصيل الطرود، الهدايا، المستندات، المشتريات، وطلبات المطاعم
- تصفح المتاجر والمطاعم وعرض المنتجات
- إتمام عملية الشراء خطوة بخطوة
- تتبع الطلب مباشر (Live Tracking)
- إدارة الملف الشخصي وسجل الطلبات

## ❌ المشاكل والثغرات الحالية (مهم جداً)

### 1. تكرار الملفات (Duplicates)
- `RestaurantsPage.tsx` و `Restaurants/page.tsx` — نفس الكود بالكامل
- `StoreDetailsPage.tsx` و `StoreDetails/page.tsx` — نفس الكود بالكامل
- `ProductDetailsPage.tsx` و `ProductDetailsPage/page.tsx`

### 2. مسارات التنقل معطلة (Broken Navigation)
| من | إلى | المشكلة |
|----|-----|---------|
| `OrderSummary/page.tsx` | `/(Screen)/PaymentMethods` | المسار خطأ — الصحيح `/PaymentMethods` |
| `PaymentMethods` | `OrderConfirmed` | لا يوجد مسار `/OrderConfirmed` كـ top-level route |
| بطاقة الخدمات في الرئيسية | كلها تذهب إلى `/Restaurants` | يجب أن تذهب لصفحات مختلفة حسب الخدمة |
| كل المطاعم | `/StoreDetails` | المسار ثابت — لا يوجد `id` ديناميكي |
| زر إضافة للسلة | لا شيء | لا يوجد تكامل مع Cart |

### 3. إدارة الحالة (State Management) — غائبة تماماً
- **السلة محلية** في `Cart/page.tsx` فقط — لا يمكن مشاركة البيانات بين الصفحات
- عند إضافة منتج من `ProductDetailsPage`، لا يظهر في السلة فعلياً
- لا يوجد **React Context** أو **Zustand** أو أي أداة حالة عامة
- تفضيلات المستخدم غير محفوظة (لا تخزين محلي)

### 4. جميع البيانات Hardcoded
- المطاعم والمتاجر والمنتجات كلها ثابتة في الكود
- لا يوجد اتصال بأي API
- الصور تستخدم روابط Googleusercontent (غير مستقرة)

### 5. ملف `next.config.ts` — إعدادات صور غير كافية
- فقط `lh3.googleusercontent.com` مسموح به
- أي صور أخرى ستفشل

### 6. Tailwind CSS — إصدار v4 مع إعدادات v3
- `globals.css` يستورد Tailwind v4 (`@import "tailwindcss"`)
- لكن `tailwind.config.js` لا يزال على نمط v3
- يمكن تحسين استخدام `@theme` بدلاً من `theme.extend`

### 7. `ClientWrapper` مستورد ولكن غير مستخدم
- في `layout.tsx` يتم استيراد `ClientWrapper`
- لكنه لا يستخدم أبداً في JSX
- على الأرجح كان من المفترض أن يدير شاشة Splash

### 8. Animation Cleanup مفقود
- `LiveTrackingPage.tsx` يستخدم `requestAnimationFrame` بدون إلغاء عند الفك (unmount)

### 9. لا يوجد Error Boundaries
- أي خطأ في أي مكون سيكسر التطبيق بالكامل

### 10. لا يوجد تخزين محلي (Local Storage / Persistence)
- عند إعادة تحميل الصفحة، تفقد كل البيانات (السلة، تفضيلات المستخدم، إلخ)

## 🛠 الأدوات الموصى بها

### إدارة الحالة
| الأداة | السبب |
|--------|-------|
| **Zustand** | بسيطة، خفيفة، تدعم middleware، مثالية لمشروع بحجمنا |
| أو React Context | مدمج مع React، لا يحتاج مكتبة إضافية (لكن أقل أداءً) |

### API Calls
| الأداة | السبب |
|--------|-------|
| **TanStack Query (React Query)** | أشهر مكتبة لإدارة البيانات غير المتزامنة، caching، refetching |
| **Axios** | HTTP client أفضل من fetch مع interceptor |

### المصادقة
| الأداة | السبب |
|--------|-------|
| **NextAuth.js (Auth.js)** | متكامل مع Next.js، يدعم Google، GitHub، وغيرها |
| أو **Supabase Auth** | حل كامل مع قاعدة بيانات |

### قاعدة البيانات (Backend)
| الأداة | السبب |
|--------|-------|
| **Supabase** | حل مفتوح المصدر، PostgreSQL، Realtime، Storage، Auth |
| أو **Firebase** | جاهز وسريع، لكن مملوك لـ Google |

### التخزين المحلي
| الأداة | السبب |
|--------|-------|
| **zustand/middleware (persist)** | حفظ تلقائي في localStorage مع أقل كود |
| **localForage** | IndexedDB wrapper للحجم الكبير |

### الخرائط
| الأداة | السبب |
|--------|-------|
| **Google Maps API** | دقة عالية، يدعم المنطقة |
| أو **Mapbox** | أفضل أداء وأسعار مرنة |

### الإشعارات
| الأداة | السبب |
|--------|-------|
| **Firebase Cloud Messaging (FCM)** | المعيار الذهبي للإشعارات |

### Testing
